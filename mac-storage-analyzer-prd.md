## 🎯 一、产品概述

### 1.1 产品定位
SpaceBadger 是一款专为 Mac 用户设计的**存储空间分析工具**，帮助用户直观了解磁盘空间分布，并通过历史对比发现存储增长趋势，**不执行任何清理操作**，将删除决策权完全交给用户。

### 1.2 核心价值主张
> "看清存储，掌控增长" - 让用户清晰知道磁盘空间去了哪里，以及是什么在持续占用空间

### 1.3 目标用户画像

| 用户类型 | 特征描述 | 核心痛点 |
|----------|----------|----------|
| 开发者 | 项目多、node_modules/依赖庞大 | 不知道哪个项目占用最多空间 |
| 设计师 | 大量素材、工程文件 | 难以追踪哪些项目在持续膨胀 |
| 普通用户 | 对技术了解有限 | 系统提示空间不足，却不知删什么 |
| 数据分析师 | 需要管理大量数据集 | 需要定期检查哪些数据集在增长 |

## 🔬 二、技术路径决策

### 2.1 技术选型对比

| 维度 | Node.js + Electron | Java + JavaFX |
|------|-------------------|---------------|
| **开发效率** | ⭐⭐⭐⭐⭐ 快速迭代 | ⭐⭐⭐ 周期较长 |
| **可视化生态** | ⭐⭐⭐⭐⭐ D3.js/ECharts丰富 | ⭐⭐⭐ 需手动实现较多 |
| **Mac适配** | ⭐⭐⭐⭐⭐ 原生支持良好 | ⭐⭐⭐⭐ 需要额外配置 |
| **性能** | ⭐⭐⭐ 需优化内存 | ⭐⭐⭐⭐⭐ 原生性能好 |
| **安装包大小** | ⭐⭐ ~150MB | ⭐⭐⭐⭐ ~50MB |
| **社区支持** | ⭐⭐⭐⭐⭐ 活跃 | ⭐⭐⭐ 相对小众 |
| **参考项目** | space-radar、其他开源 | JDiskReport |

### 2.2 决策结论

**✅ 推荐选择：Node.js + Electron**

**理由：**
1. **可视化是核心差异化特性** - D3.js/ECharts生态可快速实现高质量的交互式图表
2. **快速验证产品假设** - 能在短周期内完成MVP并收集用户反馈
3. **现代UI体验** - 可利用React/Vue构建流畅的用户界面
4. **参考项目丰富** - space-radar等开源项目提供技术参考
5. **多线程扫描** - 可使用Worker Threads解决性能问题

**性能优化策略：**
- 使用 Worker Threads 进行后台扫描，避免UI阻塞
- 增量渲染大型目录结构
- 延迟加载深层目录
- 本地SQLite存储扫描结果

## 💾 六、数据模型

### 6.1 核心数据结构

```typescript
// 扫描快照
interface Snapshot {
  id: string;                    // UUID
  name: string;                  // 用户自定义名称（可选）
  createdAt: Date;               // 创建时间
  scanPath: string;              // 扫描根路径
  totalSize: number;             // 总大小（字节）
  fileCount: number;             // 文件总数
  folderCount: number;           // 文件夹总数
  scanDuration: number;          // 扫描耗时（毫秒）
  rootNode: FolderNode;          // 目录树根节点
}

// 文件夹节点
interface FolderNode {
  path: string;                  // 完整路径
  name: string;                  // 文件夹名称
  size: number;                  // 大小（字节）
  fileCount: number;             // 直接子文件数
  children: FolderNode[];        // 子文件夹
  isAccessible: boolean;         // 是否有权限访问
}

// 对比结果
interface ComparisonResult {
  snapshotA: Snapshot;
  snapshotB: Snapshot;
  totalDiff: number;             // 总大小差异
  daysBetween: number;           // 间隔天数
  topGrowing: DiffItem[];        // 增长最多的目录
  topShrinking: DiffItem[];      // 减少最多的目录
}

// 差异项
interface DiffItem {
  path: string;
  sizeA: number;
  sizeB: number;
  diff: number;                  // sizeB - sizeA
  diffPercent: number;           // 增长百分比
  status: 'new' | 'deleted' | 'modified';
}
```

### 6.2 本地存储方案

使用 **SQLite** 存储扫描快照和用户设置：

```sql
-- 快照表
CREATE TABLE snapshots (
  id TEXT PRIMARY KEY,
  name TEXT,
  scan_path TEXT NOT NULL,
  total_size INTEGER NOT NULL,
  file_count INTEGER,
  folder_count INTEGER,
  scan_duration INTEGER,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 文件夹数据表（扁平化存储，便于查询）
CREATE TABLE folder_nodes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  snapshot_id TEXT NOT NULL,
  path TEXT NOT NULL,
  name TEXT NOT NULL,
  size INTEGER NOT NULL,
  file_count INTEGER,
  parent_path TEXT,
  depth INTEGER,
  FOREIGN KEY (snapshot_id) REFERENCES snapshots(id)
);

-- 索引优化
CREATE INDEX idx_folder_snapshot ON folder_nodes(snapshot_id);
CREATE INDEX idx_folder_path ON folder_nodes(path);
CREATE INDEX idx_folder_size ON folder_nodes(size DESC);

-- 用户设置
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT
);
```

### 7.3 关键技术点

**1. 高性能目录扫描**

```javascript
// 使用 Worker Thread 避免阻塞主进程
// scanner.worker.js
const { parentPort, workerData } = require('worker_threads');
const fs = require('fs');
const path = require('path');

async function scanDirectory(dirPath, depth = 0) {
  const stats = await fs.promises.stat(dirPath);
  const result = {
    path: dirPath,
    name: path.basename(dirPath),
    size: 0,
    children: []
  };

  if (stats.isDirectory()) {
    try {
      const entries = await fs.promises.readdir(dirPath, { withFileTypes: true });
      for (const entry of entries) {
        const childPath = path.join(dirPath, entry.name);
        if (entry.isDirectory()) {
          const child = await scanDirectory(childPath, depth + 1);
          result.children.push(child);
          result.size += child.size;
        } else {
          const childStats = await fs.promises.stat(childPath);
          result.size += childStats.size;
        }
      }
      // 定期向主进程报告进度
      if (depth < 3) {
        parentPort.postMessage({ type: 'progress', path: dirPath, size: result.size });
      }
    } catch (err) {
      result.isAccessible = false;
    }
  }
  return result;
}
```
